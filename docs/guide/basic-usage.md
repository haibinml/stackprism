# 基本使用

弹窗自上而下分五个区域。

## 顶部工具栏

```text
[栈棱镜 v1.0.x  当前页 URL]    [主题] [设置] [复制] [刷新]
```

- **左边**：扩展名 + 版本号 + 当前页 URL（截断，hover 显示完整）
- **右边四个图标按钮**：
  - **主题**（太阳 / 月亮 / 屏幕图标）：点击在「自动 / 浅色 / 深色」三个状态间循环
  - **设置**：打开设置页（独立标签页）
  - **复制**：把当前检测的完整 raw JSON 复制到剪贴板
  - **刷新**（绿色实色按钮，主操作）：通知后台重新检测当前页面

## 概览数据

```text
22 技术    274 资源    0 响应头
```

数字会带 count-up 动画。

- **技术**：检测到的去重技术总数
- **资源**：当前页面加载过的脚本 / 样式 / iframe / 图片等资源 URL 数量
- **响应头**：从主文档响应头里提取出的有用 header 数量

## 分类过滤栏

```text
[ 重点 5  全部 22 ]    [ 选择分类 ▾ ]
```

- 左侧是 segment 切换器，「重点」只列高置信度结果，「全部」列全部
- 右侧是分类下拉，22 个分类（前端框架 / UI 框架 / CDN / 网站程序 / ...），选具体某个分类只显示该分类的结果。每个选项后面带计数（如「网站程序 · 4」）

## 技术列表

每条技术一个卡片：

```text
React                                         [高置信度]
- DOM 节点存在 React Fiber 标记
- 资源 URL 匹配 cdn.example.com/react.production.min.js
来源：响应头  动态监控
识别不准确，点击纠正
```

- **技术名**：可点击，跳转到该技术的官网或仓库（鼠标悬停时右侧出现外链图标）
- **置信度药丸**：高（绿）/ 中（黄）/ 低（灰）
- **证据**：最多 4 行，列出让插件得出这个判断的 DOM / 资源 / 响应头片段
- **来源**：每个标签可点击，**点击会在底部弹出一个抽屉，只显示这条技术、这条来源的原始 JSON 数据**
- **纠正按钮**：默认透明，hover 卡片时渐显。点击会打开 GitHub Issue 表单并自动填好这条识别结果，方便反馈误报 / 漏报

::: tip 性能优化
列表项启用了 `content-visibility: auto`，屏幕外条目浏览器跳过 layout/paint，几百条技术也不会卡。
:::

## 底部工具栏

```text
[搜索]  [原始线索]                                  GitHub
```

左侧两个工具按钮，详情见 [辅助工具](./tools.md)。点击后会从底部滑出抽屉式面板，再点同个按钮 / 点抽屉里的 X 关闭。

## 滚动行为

弹窗整体高度锁定 600px，只有**技术列表区域**自己滚动。顶部工具栏、概览、分类过滤栏始终可见。

列表向下滚超过约 240px 时右下角浮出一个圆形按钮（向上箭头），点击平滑滚回顶部。
